## 원시값
1. 원시 값은 변경불가한 값이다.\
변수는 하나의 값을 저장하기위한 메모리 저장의 주소이고 값은 그 변수에 저장된 데이터 이므로 값이 변경 불가한 것이지 변수가 변경불가능한 것은 아니다. 변수는 재할당을 통해 변수의 값이 변경될 수 있다.
2. 문자열과 불변성\
자바스크립트의 문자열은 원시탑이며 변경 불가이다.\
아래의 코드를 실행하면 'Hi'와 'Hello'가 모두 메모리에 존재하고 식별자인 str의 메모리 주소는 'Hi'의 주소를 나타내다 두번째 줄이 실행되면서 'Hello'의 주소를 나타내게 된다.
```js
let str = 'Hi'
str = 'Hello'
```
3. 값에 의한 전달
```js
let score = 80;
let cod = score;
console.log(score) //80
console.log(cod) //80

score = 100;

console.log(score)//100
console.log(cod) //80
```

변수에 원시값을 가진 변수를 할당하면 원시값이 복사되어 전달 되는데 이를 값에 의한 전달이라고 한다. 위 코드를 보면 80이라는 숫자는 같지만 둘이 별개의 메모리 공간에 저장되기 때문에 score에 100을 재할당하더라도 cod에는 영향을 주지 않는다.

## 객체 
- 객체는 원시값과 다른 방식으로 동작하게 되있다.
1. 변경 가능한 값\
객체는 할당된 변수가 가진 메모리 주소를 통해 메모리공간에 접근하여 참조 값에 접근한다. 원시값은 변경이 불가능해서 재할당을 해야지만 변경이 가능했지만 객체는 재할당없이 직접 변경할 수 있다.이러한 특성때문에 여러개의 식별자가 하나의 객체를 공유할 수 있다는 단점이 있다.

2. 참조에 의한 전달\
객체를 가리키는 변수를 다른변수에 할당하면 참조 값이 복사되어 전달된다.
이 경우 두개의 식별자가 하나의 객체를 공유하는 것으로 객체를 변경하면 두 식별자 모두 영향을 받는다.
